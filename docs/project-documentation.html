4. Autenticación

LDAP / Active Directory

  • LdapService sincroniza usuarios desde AD
  • Botón "Sincronizar AD" en /admin/users
  • Mapea campos LDAP → modelo User
  • Crea nuevos o actualiza existentes por email

Laravel Fortify

  • Login/logout estándar
  • Soporte 2FA (TOTP) con QR
  • Recovery codes disponibles
  • Modal de configuración en el frontend

Selección en cascada (AltaCursoModal)

Al crear un curso, el admin asigna CDCs mediante un flujo de 3 pasos:

1
Seleccionar Área — dropdown con todas las áreas. Muestra el saldo disponible del presupuesto por área.
2
Seleccionar Departamento — filtrado por el área elegida. Muestra saldo del presupuesto del departamento.
3
Seleccionar CDC — filtrado por departamento (cdcs.id_departamento). Si hay un solo CDC, se auto-selecciona.
4
Definir monto — el admin indica cuánto se descuenta de ese CDC. Puede agregar múltiples CDCs.
⚠️ Prerequisito: La tabla cdcs debe estar poblada con al menos un registro por departamento para que la asignación funcione. Los CDCs se crean desde Administración → Estructura → tab CDCs. Sin CDCs cargados, el dropdown del paso 3 quedará vacío y no se podrá asignar financiamiento al curso.

12. Cómo Ejecutar el Proyecto

# Instalar dependencias
composer install
npm install

# Configurar .env (DB, LDAP, MAIL)
cp .env.example .env
php artisan key:generate

# Migraciones
php artisan migrate

# Dev server
npm run dev          # Vite (frontend)
php artisan serve    # Laravel (backend)
